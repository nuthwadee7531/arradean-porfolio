Churn Modeling in Telco Business

Overview

This project focuses on developing a predictive model to identify telco subscribers at high risk of churning. By analyzing historical contract types, monthly charges, and usage metrics, we help customer retention teams execute targeted interventions.

Methodology

  1. Cleaned and preprocessed customer demographic and usage features in Python (Pandas).
  2. Executed SQL queries to join billing datasets with subscriber interaction logs.
  3. Built and tuned classification models (XGBoost) using Scikit-Learn.

SQL Query to Isolate High-Risk Segments

-- SQL Query to isolate high-risk customer segments
SELECT 
    customer_id, 
    tenure_months, 
    contract_type, 
    monthly_charges,
    internet_service_type,
    CASE 
        WHEN monthly_charges > 70 AND tenure_months < 12 AND contract_type = 'Month-to-month' THEN 'High Risk'
        WHEN contract_type = 'Month-to-month' THEN 'Medium Risk'
        ELSE 'Low Risk'
    END AS risk_tier
FROM telco_customers
WHERE status = 'Active'
ORDER BY monthly_charges DESC;

Key Insights

  • Contract Risk: Customers on month-to-month contracts are 5x more likely to churn than those on annual contracts.
  • Friction Points: Subscribers experiencing payment method failures churned within 30 days of the failure.
  • Retention Strategy: Offered a 15% discount for switching from fiber optic to DSL/basic packages, saving 22% of at-risk users.

Regression Predicting Stock Price

Overview

Utilizing historical price data, moving averages, and volatility indices, this project implements multiple regression models to forecast short-term stock prices and support algorithmic trading decisions.

Methodology

  1. Extracted historical equity data using yfinance.
  2. Engineered technical indicators (RSI, MACD, 50-day and 200-day Simple Moving Averages).
  3. Trained and cross-validated Linear Regression and Ridge Regression models.

Customer Segmentation using Clustering

Overview

Understanding user diversity is key to targeted marketing. This project applies unsupervised learning to segment e-commerce customers into distinct groups, optimizing marketing campaign spend.

Methodology

  1. Calculated Recency, Frequency, and Monetary (RFM) values from transaction logs.
  2. Applied Log Transformation and MinMax scaling to normalize heavily skewed distributions.
  3. Compared K-Means (Elbow method/Silhouette score), Agglomerative Hierarchical, and DBSCAN clustering.

Python Portfolio Clustering Snippet

  • Optimal Segments: Identified 4 distinct customer personas: Champions, Loyal Customers, At-Risk, and Hibernating.
  • Targeted Impact: Tailored marketing campaigns directed at the 'At-Risk' cohort boosted engagement by 28% and recovered 12% in sales.
